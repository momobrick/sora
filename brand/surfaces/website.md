# Website Surface

Website surfaces translate Sora into structured digital communication. The website should explain the product clearly while preserving the refined, climate-adaptive visual world.

## Purpose

Use this file for:

- Landing page hero mockups
- Product explainer sections
- Digital UI layouts
- Website-style final project touchpoints
- Assignment 6 secondary touchpoint, if using a digital interface instead of motion

## Surface Role

The website should answer three questions quickly:

1. What is Sora?
2. How does the roof respond?
3. Why does it matter for homes and climate adaptation?

## Page Hierarchy

Recommended landing page structure:

1. **Hero:** A roof that responds to temperature.
2. **Product explanation:** Reflect Mode and Absorb Mode.
3. **How it works:** Roof surface temperature, threshold, stabilization buffer, independent zones.
4. **Use cases:** Hot climates, cold conditions, seasonal swings, mixed roof exposure.
5. **Practical adaptation:** Residential-first, accessible, scalable.
6. **Governance / accuracy note:** Sora responds to roof surface temperature only.

## Hero Section

The hero should be visually simple and product-led.

Recommended hero copy:

**A roof that responds to temperature.**  
Sora is a responsive roofing material that shifts between Reflect Mode and Absorb Mode based on roof surface temperature.

Recommended visual:

- Realistic home or roof detail
- Roof plane visible and central
- Calm environmental context
- Clear material quality
- Optional split-mode or mode-transition visual

## Website Typography

Use Marcellus for the hero headline and major section headlines. Use Inter for body copy, navigation, UI labels, captions, product explanations, mode cards, diagrams, and interface text.

The website should feel editorial and spacious, but still practical and easy to understand. Marcellus should create warmth and hierarchy; Inter should carry the functional communication.

## Product Explainer Section

Use two side-by-side cards or panels:

### Reflect Mode

Lightens when roof surface temperature rises, helping reflect heat from the roof surface.

### Absorb Mode

Darkens when roof surface temperature drops, helping the roof surface absorb warmth.

Keep the language descriptive and controlled. Do not make performance claims that the project has not substantiated.

## UI / Interface Treatment

If showing a product interface, it should feel like a quiet control and monitoring layer, not a flashy smart-home dashboard.

Appropriate UI elements:

- Mode status
- Roof zone map
- Surface temperature
- Manual override
- Sync all zones
- Threshold setting

Avoid:

- Neon graphics
- Excessive data visualizations
- AI assistant mascots
- Energy-savings claims
- Futuristic holographic controls

## Layout Style

Website surfaces should feel:

- Spacious
- Editorial
- Structured
- Calm
- Precise
- Residential, not corporate-tech

Use generous white or soft neutral space. The website should not feel like a dense engineering dashboard.

## Website Prompt Scaffold

```text
Create a clean website landing page mockup for Sora, a responsive roofing material for residential homes. The hero section should feature a realistic roof-focused image and the headline “A roof that responds to temperature.” Present Sora as practical climate adaptation, not generic smart-home technology. Include calm product explainer sections for Reflect Mode and Absorb Mode, a simple roof-zone concept, and minimal interface elements. Keep the design editorial, spacious, refined, and easy to understand. Avoid sci-fi UI, neon effects, generic sustainability clichés, solar panels, exaggerated performance claims, and decorative AI artifacts.
```

## Assignment 6 Translation Notes

Website works well as a digital touchpoint because it tests whether the system can move from atmospheric imagery into explanatory structure.

What should stay consistent:

- Product logic
- Reflect / Absorb mode language
- Material realism
- Calm climate-adaptive tone
- Residential-first framing

What changes:

- The website needs more hierarchy and explanation.
- Images may become smaller and more modular.
- Product facts need to be more explicit.
- Captions and UI labels must carry more of the communication.

## Recommended Final Project Use

Use the website surface in the Visual World section as a digital brand application. It can also support the System Logic section if showing how the product interface communicates roof zones, mode status, and threshold behavior.

## Do

- Use the headline system consistently.
- Show product logic in plain language.
- Use UI labels only where they clarify the system.
- Keep imagery realistic and restrained.
- Make the page feel useful and believable.

## Do Not

- Turn Sora into a software company.
- Make the website feel like a speculative app concept only.
- Overuse charts, icons, dashboards, or generic climate data.
- Use unverified claims about energy savings or cost reduction.
- Let interface graphics overpower the roofing material.

## Quality Check

A strong Sora website surface should feel like a credible product landing page for an adaptive building material. If it feels like a generic climate-tech startup page, reduce the abstraction and return attention to the roof.
