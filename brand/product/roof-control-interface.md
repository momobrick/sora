# Roof Control Interface

## Purpose

This file defines Sora's homeowner-facing roof control interface: what it is, what it controls, how it should feel, and how it should be represented in copy, product storytelling, and image prompts.

Use this file when describing the interior control experience or showing how a homeowner interacts with Sora.

## Core Definition

The roof control interface is a simple interior wall-mounted control point for Sora's responsive roofing system.

It allows the homeowner to view and adjust the roof's control behavior without interacting directly with the roof surface.

The interface is envisioned as similar in familiarity and placement to a furnace or air conditioning thermostat. It should feel practical, calm, and easy to understand.

## Product Role

The roof control interface is the homeowner interaction layer for Sora.

It supports the roofing system by allowing the homeowner to:

- view the current roof behavior
- select automatic independent mode
- select automatic synced mode
- temporarily use manual override mode
- understand whether the roof is in reflect mode or absorb mode

The interface should make the product feel approachable without making the homeowner responsible for constant management.

## Relationship to the Roof System

The roof remains the primary product.

The interface does not create the roof response, replace the roof response, or manually control every roof section by default. It simply gives the homeowner a clear, familiar way to understand and adjust the system when needed.

In normal use, Sora operates automatically based on roof surface temperature.

## Control Behaviors

### Automatic Independent Mode

Automatic independent mode allows each defined roof zone to respond to its own roof surface temperature.

The interface may show that different roof zones are responding independently, but it should not imply that the homeowner needs to manage each zone constantly.

Use this mode when communicating adaptive, site-specific response behavior.

### Automatic Synced Mode

Automatic synced mode allows the entire roof to respond as one unified system.

The interface may present this as a simple preference for homeowners who want a more visually unified roof behavior or a simpler whole-roof setting.

Use this mode when communicating a coordinated, roof-wide experience.

### Manual Override Mode

Manual override mode allows the homeowner to temporarily select the preferred mode, overriding automatic behavior.

Manual override may be useful for:

- personal comfort preferences
- maintenance
- unusual weather patterns
- testing
- specific household needs

Manual override should always be described as temporary and secondary to automatic behavior.

## Interface Experience Principles

The roof control interface should feel:

- familiar
- simple
- practical
- calm
- residential
- easy to understand
- integrated into home comfort routines

The interface should not feel:

- technical
- intimidating
- futuristic for its own sake
- app-first
- gadget-like
- overly automated
- like a complex smart-home control center

## Visual Guidance

When showing the interface, make it look like a believable residential wall-mounted control panel.

Preferred visual qualities:

- clean rectangular form
- simple screen or display area
- restrained physical controls or clear touch interface
- neutral residential wall setting
- soft, practical lighting
- clear but minimal mode labels
- calm, readable status information

Avoid visual qualities:

- glowing sci-fi screens
- holograms
- floating interfaces
- excessive data dashboards
- complex graphs
- neon colors
- futuristic smart-home theatrics
- handheld remote controls as the primary interface

## Interface Content

The interface may reference:

- current mode: reflect mode or absorb mode
- control behavior: automatic independent, automatic synced, or manual override
- roof zones, if shown simply
- roof surface temperature, if needed for explanation

The interface should not imply:

- the roof responds directly to the weather forecast
- the system is controlled by outdoor air temperature alone
- the homeowner has to manage every panel manually
- the roof generates electricity
- Sora replaces heating, cooling, insulation, or ventilation

## Copy Guidance

Preferred copy:

- "Choose independent or synced roof behavior."
- "Automatic mode lets the roof respond to surface temperature on its own."
- "Manual override gives homeowners temporary control when needed."
- "A familiar wall-mounted interface keeps roof behavior easy to understand."
- "The roof does the adapting. The homeowner stays in control when it matters."

Avoid copy:

- "Control every panel from inside your home."
- "Program your roof like a smart device."
- "The control panel predicts the weather and changes your roof."
- "Turn your roof into a climate-control machine."
- "Manage your home temperature from the roof screen."

## Image Prompt Guidance

When generating imagery of the interface, keep the prompt grounded in a familiar home environment.

Preferred prompt language:

> Show Sora's roof control interface as a simple wall-mounted residential control panel, similar in familiarity to a thermostat. The display shows automatic independent mode selected, with simple reflect mode and absorb mode status information. Keep the design calm, practical, minimal, and believable. Avoid sci-fi effects, holograms, glowing dashboards, or complex smart-home graphics.

For manual override:

> Show a homeowner using Sora's wall-mounted roof control interface to temporarily select manual override mode. The interface should feel familiar and easy to understand, with clear mode labels and restrained residential product design. Do not show the roof as mechanically controlled panel-by-panel.

## Product Accuracy Rules

- The roof control interface is an interior control point for the responsive roofing system.
- The roof remains the primary product; the interface is the homeowner interaction layer.
- Automatic roof behavior remains based on roof surface temperature.
- Manual override is temporary and secondary to automatic operation.
- The interface should not make Sora feel like a complicated smart-home gadget.
- The interface should not imply solar energy generation, weather prediction, HVAC replacement, or mechanical roof movement.
